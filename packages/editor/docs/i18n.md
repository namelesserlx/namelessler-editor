# I18n

`@namelesserlx/editor` includes built-in UI messages for:

- `en-US`
- `zh-CN`

Use the `locale` prop for the React editor:

```tsx
import { useEditorController } from '@namelesserlx/editor/react/controller';
import { Editor } from '@namelesserlx/editor/react/editor';

const editor = useEditorController({
  defaultContent: content,
  locale: 'zh-CN',
});

<Editor controller={editor} />;
```

Readonly rendering emits content HTML only. Keep article chrome, labels, and surrounding UI localized in the consuming application.

## Defaults

- Default locale: `en-US`
- Unknown runtime locale values fall back to `en-US`
- v1 does not expose a public `messages` override prop

Keeping overrides out of v1 keeps the public API smaller. If product requirements later need tenant-specific wording, add a focused API after real usage appears.

## What Is Localized

Built-in messages cover:

- toolbar labels
- bubble menu labels
- link popover labels
- color picker labels
- readonly empty state

## Advanced Imports

Most users should not import i18n helpers directly. They exist for advanced integration and tests:

```ts
import {
  DEFAULT_EDITOR_LOCALE,
  SUPPORTED_EDITOR_LOCALES,
  resolveEditorMessages,
} from '@namelesserlx/editor/i18n';
```
